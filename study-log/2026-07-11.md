## O que entendi

Git é uma ferramenta local de controle de versão. Ele registra o histórico de mudanças de um projeto no próprio computador.

GitHub é uma plataforma online para hospedar e sincronizar repositórios Git.

O fluxo inicial de trabalho com Git é:

git status
git add .
git commit -m "mensagem do commit"
git push

Também entendi que:

- `git init` transforma uma pasta comum em um repositório Git local.
- `git add` prepara arquivos para o próximo commit.
- `git commit` registra mudanças localmente.
- `git remote add origin` conecta o repositório local a um repositório remoto.
- `git push` envia commits locais para o GitHub.
- Git não rastreia pastas vazias; por isso usei arquivos `.gitkeep`.
