# 2026-07-14

## Sessão 1 — madrugada

### Tema

Retomada dos estudos e início do primeiro micro-projeto em Python.

### Contexto

Voltei aos estudos depois de não ter feito blocos relevantes no domingo e na segunda-feira.

A sessão começou na madrugada de terça-feira, 14/07/2026, com o objetivo de decidir se eu deveria continuar fazendo exercícios isolados ou começar um micro-projeto.

A decisão foi avançar para um micro-projeto, porque exercícios ensinam conceitos específicos, mas projetos consolidam a organização do código.

### O que foi feito

- Iniciei o micro-projeto `controle-estudos-cli`.
- Estruturei o programa com uma função principal `main()`.
- Criei um menu de terminal com opções para registrar sessão, listar sessões, mostrar resumo e sair.
- Usei um loop para manter o programa rodando até o usuário escolher encerrar.
- Criei uma lista `sessoes` para armazenar os registros em memória.
- Passei a lista `sessoes` como argumento para funções que precisam acessar os dados.
- Implementei o registro de sessões de estudo usando dicionários.
- Criei funções auxiliares para leitura e validação de campos.
- Implementei a listagem de sessões registradas.
- Fiz um commit com a estrutura inicial do micro-projeto.

### O que entendi

Entendi melhor a diferença entre exercícios isolados e um micro-projeto.

Nos exercícios, eu pratico uma peça específica da linguagem. No micro-projeto, preciso organizar várias peças ao mesmo tempo: menu, funções, estado em memória, validação, dicionários, listas e fluxo do programa.

Também entendi melhor a diferença entre parâmetro e argumento. A lista `sessoes` nasce dentro da função principal e é passada para outras funções quando elas precisam trabalhar com os dados registrados.

Outro ponto importante foi entender que algumas funções alteram o estado do programa, como a função de registrar sessão, enquanto outras apenas consultam esse estado, como a função de listar sessões.

### Conceitos praticados

- Funções
- Parâmetros e argumentos

- Listas

- `input()`

### Dificuldades

A principal dificuldade foi entender como declarar parâmetros em funções.

Também houve confusão inicial sobre como uma função definida fora do loop poderia acessar uma lista criada dentro da função principal. A solução foi passar a lista como argumento para as funções que precisam dela.

Outra dificuldade foi dosar o nível de ajuda: o objetivo é receber orientação sem receber o código pronto.

### Decisões tomadas

O projeto será desenvolvido em etapas.

Nesta primeira versão, o foco é manter os dados apenas em memória. Persistência com JSON, banco de dados e orientação a objetos ficam para versões futuras.

Também decidi que, por enquanto, o mais importante é consolidar programação procedural antes de avançar para OOP.
